## 分析

### 亮点

1. **记忆模式 J-Score 超越基线（+0.015）** — 新增的 forget API 有效防止了 `ask_to_forget` 和 `sensitive_info` 类别的隐私泄露。这是 OpenCortex 首次在 PersonaMem 上超越全文上下文基线。

2. **敏感信息 J-Score +0.169** — 所有类别中提升最大。被遗忘的敏感记忆不再出现在检索结果中，模型能够在不泄露隐私信息的情况下给出通用回答。

3. **对话模式 Recall@5 从 0.072 跃升至 0.444** — text-embedding-3-small 在对话观察检索上显著优于 BGE-M3 本地向量模型。v0.6 的 flat search 修复也有贡献。

4. **Token 压缩 96-99%** — OpenCortex 在所有模式下均实现 30-100 倍的上下文压缩，同时保持可比的准确度。

### 不足

1. **对话模式 J-Score -0.236** — gpt-5.4-nano 缺乏多跳推理深度（Cat 4: -0.275）。模型能检索到相关上下文（Recall@5=0.444），但无法有效整合信息生成答案。使用更强的回答模型（gpt-5.4-mini 或 Qwen3-32B）可以将检索质量与回答质量分离评估。

2. **文档模式未完成** — QASPER 全量 ingestion 需要约 3000-5000 次 LLM+embedding API 调用（281 篇论文）。受限的远程 API 使其不可行。建议使用本地向量模型 + 更快的 LLM 处理文档密集型评测。

3. **记忆模式 Recall@5 = 0.249** — 低于对话模式（0.444）。PersonaMem 的属性是简短且语义相似的陈述，仅靠向量相似度难以区分。`健康与医疗` 类别的 Recall@5 最低（0.100）。

### Forget 功能效果

forget API（commit 554b4a0）在写入 `ask_to_forget` 属性后立即删除，防止检索泄露。

| 类别 | 加 forget 前 J-Score | 加 forget 后 J-Score | 差值 |
|------|---------------------|---------------------|------|
| 要求遗忘（ask_to_forget） | 0.200（快速测试） | **0.621**（全量） | **+0.421** |
| 敏感信息（sensitive_info） | — | **0.484**（+0.169 vs 基线） | — |

### 优化建议

1. **回答模型与服务端模型分离** — 服务端任务（embedding、摘要生成、意图分析）使用 gpt-5.4-nano，但评测的回答生成和评判使用更强的模型（gpt-5.4-mini），以准确评估检索质量。
2. **文档模式使用本地向量** — 切回本地 BGE-M3 或使用批量 embedding API，避免文档 ingestion 时的限流瓶颈。
3. **提升记忆检索能力** — PersonaMem Recall@5=0.249 表明仅依赖向量相似度不足以区分短偏好文本。可考虑关键词增强或类别感知检索。
