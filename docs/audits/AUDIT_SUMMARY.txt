================================================================================
OPENCORTEX AUDIT: STORE & RECALL PATH CONSISTENCY SUMMARY
================================================================================

AUDIT SCOPE
-----------
Three ingest modes: memory, document, conversation (immediate + merged)
Focus: fact_point and anchor_projection derived records consistency

KEY FINDINGS
============

1. OVERALL CONSISTENCY: 85%
   - Write paths properly converge at _sync_anchor_projection_records()
   - Three-layer search correctly filters all modes
   - Scope field inheritance working

2. CRITICAL GAPS (P0): 2 Issues
   ✗ Conversation immediate mode has NO fact_points (by design per R23)
     - Trade-off: speed over retrieval quality
     - Intentional but creates immediate/merged asymmetry
   
   ✗ Immediate + merged records have different retrieval surfaces
     - Immediate: anchors only
     - Merged: anchors + fact_points
     - Conversation history not uniformly enriched

3. HIGH SEVERITY BUGS (P1): 2 Issues
   
   🔴 BUG ADV-001: update() does NOT regenerate fact_points
      Location: orchestrator.py, update() method [L2304-2470]
      Root cause: fact_points field never injected into abstract_json
      Impact: Edited memories lose fact_point derivations, degrade in quality
      
      Expected behavior: When content changes, _derive_layers() is called [L2370]
                         but fact_points result is discarded
      
      Proposed fix: Inject derive_result.get("fact_points", []) into 
                    abstract_json before _sync_anchor_projection_records()
   
   ⚠️  Directory nodes (document mode) skip derivation
      Location: _sync_anchor_projection_records() [L1737-1738]
      Issue: is_leaf=False records don't get anchors/fp
      Status: Likely intentional (directories are structural containers)
      Action: Verify design intent, add clarifying comment

4. MEDIUM RISKS (P2): 5 Issues
   - Scope field inheritance pattern not explicit (fragile, works now)
   - Batch_add ingest_mode marking ambiguous (likely OK)
   - Fact_point quality gate applied twice (defense-in-depth, OK)
   - Immediate records can't be enriched later (design choice)

WRITE PATH CONSISTENCY
======================

Memory Mode:
  ✓ add() → _derive_layers() → fact_points injected → _sync_anchor_projection_records()
  ✓ Both anchor + fact_point records generated
  ⚠️ update() path broken (P1 bug)

Document Mode:
  ✓ add() → _add_document() → per-chunk add() → normal memory flow
  ✓ Each leaf chunk: fact_points injected, both record types generated
  ⚠️ Directory nodes: skip derivation (intentional)

Conversation Immediate:
  ✓ _write_immediate() → record write → _sync_anchor_projection_records()
  ✗ NO _derive_layers() call (intentional per R23)
  ✗ NO fact_points generated (design trade-off)
  ✓ Anchor records only

Conversation Merged:
  ✓ _merge_buffer() → add() → _derive_layers() → fact_points injected
  ✓ Both anchor + fact_point records generated
  ✓ Converges to memory mode path

READ PATH CONSISTENCY
=====================

Three-layer search working correctly across all modes:
  ✓ Layer 1: is_leaf=True records (retrieval_surface="l0_object")
  ✓ Layer 2: anchor_projection records (retrieval_surface="anchor_projection")
  ✓ Layer 3: fact_point records (retrieval_surface="fact_point")

Scope filters per mode:
  ✓ Memory: parent_uri filter (empty session_id, source_doc_id)
  ✓ Document: source_doc_id filter (inherited by chunks and derived records)
  ✓ Conversation: session_id filter (inherited by all records and derived records)
